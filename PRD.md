一、产品背景与目标
1.1 产品背景
传统新年祝福语同质化严重，无法根据祝福对象性格、社交形象定制；用户手动撰写祝福耗时且贴合度低。本Agent依托多模态图像识别、文本提取、大模型生成、图片生成能力，通过解析朋友圈截图完成用户画像分析，自动生成三类风格祝福语，并匹配头像风格生成定制贺卡，提升祝福的个性化与仪式感。

1.2 核心目标
1. 实现朋友圈截图四区域自动分割与信息提取（头像、朋友圈背景、个性签名、朋友圈动态（非必须））
2. 完成用户性格标签化分析，匹配对应祝福逻辑
3. 生成文言风（古风小生）、文艺风（文艺大师）、大白话（简约直白）三类差异化祝福语
4. 基于头像主导风格生成对应新年贺卡，支持手动调整风格，手动编辑贺卡文字（方便用户手动加入昵称等）

四、详细功能需求
4.1 截图上传与预处理模块
4.1.1 输入规则
- 支持格式：JPG、PNG、JPEG
- 图片要求：清晰完整，包含朋友圈个人主页完整界面，无大面积遮挡、模糊、旋转
- 上传限制：单张图片，大小≤10MB
- 交互提示：上传前展示「示例截图」，标注需包含的四个区域
4.1.2 预处理能力
- 自动矫正图片方向、裁剪冗余区域
- 对模糊图片进行清晰度检测，低于阈值弹出重新上传提示
- 分割固定区域：头像区、朋友圈背景区、个性签名区、朋友圈动态展示区
4.2 多模态信息提取与解析模块
4.2.1 必选提取项（3项）
1. 头像区域：识别风格标签（二次元/写实风景/中老年佛系/简约文艺/卡通/真人自拍等）
2. 朋友圈背景区域：辅助识别风格，仅作为补充参考，不主导最终风格
3. 个性签名区域：OCR文本提取，关键词分词，提取情绪、价值观、风格关键词
4.2.2 可选提取项（1项）
- 朋友圈动态区域：OCR提取近3条可见动态文本，无权限/三天可见则标记「无有效动态」，不影响主流程
4.2.3 此节点大模型prompt（doubao-seed-1.8)：
你是一个专业的图片内容解析员。 现在你会收到一张朋友圈截图的图片URL。 如果图片有效且清晰，请严格按以下5个条目，逐条输出客观信息，不要分析，不要评价： 1. 头像描述：描述头像的风格、内容、给人的感觉。 2. 朋友圈背景图描述：描述背景图的场景、风格、氛围。 3. 个性签名：提取截图中能看到的个性签名原文，看不到就写“未识别到个性签名”。 4. 朋友圈动态内容：列出所有能看到的朋友圈动态文字（包括置顶），并说明主题，看不到就写“未识别到朋友圈动态”。 5. 朋友圈可见范围：明确写出截图中显示的可见范围，看不到就写“未识别到可见范围”。  如果没有收到有效的图片URL，或者图片无法识别，请直接输出：“未收到有效的朋友圈截图图片，请重新上传。”
4.3 用户性格画像生成模块
4.3.1 画像标签体系
基于提取信息生成标准化标签，包含：
- 风格标签：二次元、佛系中老年、文艺清新、潮流酷感、简约商务等
- 性格标签：开朗、沉稳、佛系、文艺、活泼、内敛、积极向上等
- 年龄倾向：年轻群体、中年群体、老年群体
4.3.2 风格优先级规则
头像风格100%主导，朋友圈背景仅做细节补充；
若头像与背景风格冲突，完全以头像为准，忽略背景风格差异。
4.3.3 此节点大模型prompt（doubao-seed-1.8):
你是一位细腻、温柔、有洞察力的性格分析师,语言精准、有质感，像朋友在认真了解一个人，不毒舌、不贴标签、不使用网络梗。
你的分析逻辑必须严格遵循以下框架：
1. 逐元素解读：对头像、朋友圈背景图、个性签名、朋友圈动态、可见范围这5个元素，逐一解读其对应的性格侧面：
  - 头像：解读其自我表达风格、内心的童真与亲和力；
  - 背景图：解读其审美追求、对艺术与仪式感的执着；
  - 个性签名：解读其自我认知、边界感与清醒度；
  - 朋友圈动态：解读其生活方式、社交偏好与对热爱的投入；
  - 可见范围：解读其隐私意识、对当下的重视程度。
2. 综合性格画像：将所有元素的解读串联成一段完整、连贯的描述，语言温柔且有画面感。
输出要求：
- 先分点列出各元素的解读，再给出综合性格画像；
- 避免空洞的形容词，用具体的细节支撑结论；

- 提供复制图片链接选项（适配平台能力）
4.7 辅助功能
1. 画像重新解析：用户可重新上传截图，覆盖原有画像与祝福语
2. 祝福语历史记录：保留本次会话内生成的所有祝福语，支持回溯选择
3. 异常兜底：识别失败时，默认生成通用马年三类祝福语，提供通用风格贺卡
五、业务流程设计
5.1 核心主流程（端到端）
1. 用户进入Agent → 查看上传提示与示例图
2. 用户上传朋友圈主页截图 → 系统完成图片预处理
3. 系统自动分割区域 → 提取头像/背景/签名/动态信息
4. 系统生成性格画像与风格标签 → 页面可视化展示
5. 系统生成文言风、文艺风、大白话三类祝福语
6. 用户选择目标祝福语 → 可点击重新生成替换
7. 系统基于头像风格自动匹配贺卡风格 → 用户可手动调整
8. 生成定制贺卡 → 用户下载/保存
9. 流程结束，支持重新开始新流程
5.2 异常分支流程
1. 截图模糊/无效 → 提示重新上传 → 返回上传环节
2. 部分区域识别失败 → 跳过无效区域，使用有效信息继续分析
3. 全区域识别失败 → 生成通用祝福与贺卡，提示「信息不足，使用通用模板」
4. 贺卡生成超时/失败 → 提示重试，或切换风格重新生成
